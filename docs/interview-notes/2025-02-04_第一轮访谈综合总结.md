## 二、跨领域核心发现

### 2.1 实体分类共性

所有三个领域都确认需要**多维分类体系**：

| 领域 | 分类维度1 | 分类维度2 | 分类维度3 |
|-----|----------|----------|----------|
| **R&D** | 结构类型 | 开发阶段 | 治疗领域 |
| **供应链** | 物料类型 | 供应链阶段 | 管控等级 |
| **监管合规** | 申报类型 | 申报阶段 | 申报地区 |

### 2.2 关系复杂性共识

所有领域都确认关系具有**多维度复杂性**：

| 复杂性类型 | R&D体现 | 供应链体现 | 监管体现 |
|-----------|---------|-----------|---------|
| **上下文依赖** | 实验条件 | 供应能力 | 法规环境 |
| **时间维度** | 开发阶段 | 交货周期 | 审评周期 |
| **数值属性** | 活性值+单位 | 绩效评分 | 严重程度 |
| **因果关系** | 作用机制 | 短缺传播 | 因果评估 |

### 2.3 跨领域关联需求

| 关联关系 | 关联强度 | 关联方式 |
|---------|---------|---------|
| **R&D ↔ 监管合规** | 🔴 深度关联 | 安全性信号、证据链、风险评估 |
| **R&D ↔ 供应链** | 🟡 部分关联 | API供应商、工艺转移 |
| **供应链 ↔ 监管合规** | 🟡 部分关联 | 供应商资质、短缺报告 |

## 四、核心关系类型汇总（跨领域）

### 4.1 R&D领域关系

```
Compound ──[inhibits|activates|binds_to|regulates|degrades]──▶ Target
    ├── activity_value (数值)
    ├── activity_unit (单位)
    ├── activity_type (IC50/EC50/Kd/Ki)
    └── context (实验条件)

Target ──[participates_in|regulates]──▶ Pathway
Disease ──[associated_with]──▶ Target
Compound ──[treats|prevents]──▶ Disease
Assay ──[measures]──▶ Activity
```

### 4.2 供应链领域关系

```
Supplier ──[supplies]──▶ Material
    ├── supply_percentage (供应比例)
    └── contract_type (合同类型)

Manufacturer ──[produces]──▶ Product
    ├── capacity_volume (产能)
    └── quality_rating (质量评分)

Material ──[required_for]──▶ Product
    ├── dosage_amount (剂量)
    └── quality_spec (质量标准)

ShortageEvent ──[affects]──▶ Drug
    ├── severity (严重程度)
    └── expected_duration (持续时间)

ShortageEvent ──[caused_by]──▶ Cause
ShortageEvent ──[propagates_to]──▶ ShortageEvent
```

### 4.3 监管合规领域关系

```
Company ──[submits]──▶ Submission
Submission ──[submitted_to]──▶ RegulatoryAuthority
RegulatoryAuthority ──[makes_decision]──▶ Submission
    ├── decision_type (Approval/CRL/Refusal)
    └── conditions_attached (附加条件)

Patient ──[experienced]──▶ SafetyEvent
SafetyEvent ──[associated_with]──▶ DrugProduct
SafetyEvent ──[assessed_by]──▶ CausalityAssessment
SafetyEvent ──[requires_reporting]──▶ RegulatoryReport

Facility ──[inspected_by]──▶ Inspection
Inspection ──[results_in]──▶ InspectionFinding
InspectionFinding ──[requires_action]──▶ CAPA
```

### 4.4 跨领域桥接关系

```
R&D ↔ 监管合规:
├── Target ──[has_safety_signal]──▶ SafetyEvent
├── Compound ──[becomes]──▶ RegulatoryDrugProduct
├── Study ──[generates_evidence]──▶ Submission
└── PreclinicalStudy ──[identifies_risk]──▶ SafetyConcern

R&D ↔ 供应链:
├── Compound(API) ──[sourced_from]──▶ Supplier
├── ManufacturingProcess ──[requires_facility]──▶ Facility
└── PCC ──[requires_sourcing]──▶ Manufacturer

供应链 ↔ 监管合规:
├── Manufacturer ──[has_certification]──▶ Certification
├── ShortageEvent ──[reported_to]──▶ RegulatoryAuthority
└── Supplier ──[compliance_status]──▶ ComplianceRecord
```

## 六、Schema设计原则总结

### 6.1 设计原则

1. **多维分类**: 所有实体采用多维度分类体系
2. **关系富化**: 关系带数值属性、上下文、时间戳
3. **状态管理**: 实体状态通过状态机建模
4. **溯源追踪**: 所有关系带来源、置信度、证据类型
5. **跨域关联**: 通过桥接实体实现跨领域关联

### 6.2 建模模式

```
实体建模模式:
├── 多维度分类: 通过多个is_a关系实现
├── 状态分离: 实体与状态分离，通过has_state关联
├── 属性分组: 核心属性、扩展属性、临时属性分组
└── 多标识符: 主标识符 + 次要标识符 + 内部标识符

关系建模模式:
├── 谓词细化: 避免泛化谓词(如has_relationship)
├── 关系属性: 数值、单位、上下文、时间戳
├── 关系约束: 基数性、传递性、对称性
└── 证据溯源: source, confidence, evidence_type
```

### 6.3 命名规范

```
实体命名:
├── PascalCase: Compound, Target, Disease
├── 组合命名: SmallMolecule, RegulatoryAuthority
└── 缩写规范: API (Active Pharmaceutical Ingredient)

关系命名:
├── 动词形式: inhibits, activates, treats, supplies
├── 动词_名词: has_activity_value, located_in
├── 方向性: uses_reserved_term → is_reserved_by
└── 传递性: part_of (传递), interacts_with (非传递)

属性命名:
├── snake_case: activity_value, approval_date
├── 单位后缀: _value, _unit, _type
└── 布尔前缀: is_, has_, can_
```

## 七、实施优先级建议

### 7.1 Phase 1: 核心本体设计（2-3周）

**优先级1 - R&D核心实体**:
```
Compound (多维分类)
Target (多维分类)
Disease
Activity (带上下文属性)
```

**优先级2 - 核心关系**:
```
Compound -[inhibits/activits/binds]-> Target
Target -[associated_with]-> Disease
Compound -[treats]-> Disease
```

### 7.2 Phase 2: 关系语义细化（2-3周）

**关系属性定义**:
- 活性关系: activity_value, unit, type, context
- 安全性关系: severity, causality, expectedness
- 供应关系: percentage, contract, quality

**关系约束定义**:
- 基数性: 1-to-1, 1-to-N, N-to-M
- 传递性: transitive, non-transitive
- 对称性: symmetric, asymmetric

### 7.3 Phase 3: 跨领域集成（3-4周）

**桥接实体设计**:
- API作为R&D-供应链桥接
- SafetyEvent作为R&D-监管桥接
- Certification作为供应链-监管桥接

**标识符映射服务**:
- Compound: InChIKey ↔ 内部编码 ↔ UNII
- Target: UniProt ↔ HGNC ↔ Entrez
- Company: DUNS ↔ FEI ↔ 内部编号
