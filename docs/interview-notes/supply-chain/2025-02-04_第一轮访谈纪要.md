## 一、核心实体确认

用户确认供应链领域需要包含以下**四类核心实体**：

| 实体类别 | 包含内容 | 优先级 |
|---------|---------|-------|
| **制造商/生产设施** | 原料药厂商、制剂厂商、CDMO等 | 高 |
| **供应商** | 原料供应商、辅料供应商、包装材料供应商 | 高 |
| **物料/产品** | 原料药(API)、辅料、包装材料 | 高 |
| **供应状态/事件** | 药品短缺、库存水平、物流状态 | 高 |

---

## 二、核心业务流程

| 流程 | 描述 | 在KG中的体现 |
|-----|------|-------------|
| **采购管理** | 从采购需求到入库的全流程 | PurchaseOrder → Supplier → Material → Receipt |
| **生产管理** | 生产计划与执行、质量控制 | Manufacturer → ProductionBatch → QualityCheck |
| **库存管理** | 库存监控、补货策略 | Material → InventoryLevel → ReorderPoint |
| **物流配送** | 运输、仓储、配送 | Shipment → LogisticsProvider → Facility → Delivery |

## 四、实体分类体系设计

### 4.1 物料/产品分类（多维）

```
Material (物料)
├── 按物料类型分类:
│   ├── API (Active Pharmaceutical Ingredient)
│   │   ├── SmallMoleculeAPI
│   │   └── BiologicAPI
│   ├── Excipient (辅料)
│   │   ├── Filler (填充剂)
│   │   ├── Binder (粘合剂)
│   │   ├── Coating (包衣)
│   │   └── Solvent (溶剂)
│   └── PackagingMaterial (包装材料)
│       ├── PrimaryPackaging (一级包装)
│       └── SecondaryPackaging (二级包装)
│
├── 按供应链阶段分类:
│   ├── RawMaterial (原料)
│   ├── Intermediate (中间体)
│   ├── BulkProduct (散装产品)
│   └── FinishedProduct (成品)
│
├── 按管控等级分类:
│   ├── ControlledSubstance (管制物质)
│   ├── CriticalMaterial (关键物料)
│   └── GeneralMaterial (一般物料)
│
└── 按储存条件分类:
    ├── ColdChain (冷链)
    ├── RoomTemperature (常温)
    └── SpecialStorage (特殊储存)
```

### 4.2 制造商/供应商分类

```
Manufacturer (制造商)
├── 按类型分类:
│   ├── APIManufacturer (原料药制造商)
│   ├── FinishedProductManufacturer (制剂制造商)
│   ├── CDMO (合同研发生产组织)
│   └── CMO (合同生产组织)
│
├── 按地域分类:
│   ├── DomesticManufacturer (国内制造商)
│   └── InternationalManufacturer (国际制造商)
│
└── 按认证状态分类:
    ├── GMP Certified (GMP认证)
    ├── FDA Approved (FDA批准)
    ├── EUDRAF Certified (欧盟认证)
    └── Pending Certification (认证中)
```

### 4.3 供应商分类

```
Supplier (供应商)
├── 按供应物料类型:
│   ├── APSupplier (API供应商)
│   ├── ExcipientSupplier (辅料供应商)
│   └── PackagingSupplier (包装材料供应商)
│
├── 按合作模式:
│   ├── SoleSupplier (独家供应商)
│   ├── PreferredSupplier (优先供应商)
│   └── BackupSupplier (备选供应商)
│
└── 按绩效等级:
    ├── Tier1 (一级供应商)
    ├── Tier2 (二级供应商)
    └── Tier3 (三级供应商)
```

### 4.4 供应状态/事件分类

```
SupplyEvent (供应事件)
├── ShortageEvent (短缺事件)
│   ├── CriticalShortage (严重短缺)
│   ├── ModerateShortage (中度短缺)
│   └── MinorShortage (轻微短缺)
│
├── QualityEvent (质量事件)
│   ├── OOS (Out of Specification)
│   ├── Deviation (偏差)
│   └── Recall (召回)
│
├── LogisticsEvent (物流事件)
│   ├── Delay (延误)
│   ├── Damage (损坏)
│   └── Loss (丢失)
│
└── RegulatoryEvent (监管事件)
    ├── WarningLetter (警告信)
    ├── ImportAlert (进口预警)
    └── ComplianceAction (合规行动)
```

## 五、关系类型设计

### 5.1 供应链基础关系

```
Supplier ──[supplies]──────────▶ Material
Manufacturer ──[produces]─────▶ Product
Facility ──[stores]────────────▶ Inventory
Shipment ──[transports]───────▶ Material
```

### 5.2 制造商能力关系（多维度）

```
Manufacturer ──[has_production_capability]──▶ Material
    ├── capacity_volume (产能数量)
    ├── lead_time (交期)
    ├── quality_rating (质量评分)
    └── cost_position (成本水平)

Manufacturer ──[has_certification]─────────▶ Certification
    ├── certification_type (GMP, FDA, etc.)
    ├── certification_number (证书编号)
    ├── issue_date (颁发日期)
    └── expiry_date (过期日期)

Manufacturer ──[collaboration_mode]───────▶ Company
    ├── OEM (原始设备制造)
    ├── CDMO (合同研发生产)
    ├── StrategicPartnership (战略伙伴)
    └── Transactional (交易关系)
```

### 5.3 短缺事件关系

```
ShortageEvent ──[affects]──────────────▶ Drug
    ├── severity (严重程度)
    ├── expected_duration (预计持续时间)
    └── affected_markets (影响市场)

ShortageEvent ──[caused_by]────────────▶ Cause
    ├── quality_issue (质量问题)
    ├── capacity_constraint (产能限制)
    ├── logistics_disruption (物流中断)
    ├── regulatory_issue (监管问题)
    └── raw_material_shortage (原料短缺)

ShortageEvent ──[impacts]─────────────▶ Stakeholder
    ├── manufacturer_impact (制造商影响)
    ├── patient_impact (患者影响)
    └── economic_impact (经济影响)

ShortageEvent ──[propagates_to]────────▶ ShortageEvent
    ├── propagation_path (传播路径)
    └── propagation_delay (传播延迟)
```

### 5.4 供应链追溯关系

```
Drug ──[contains]─────────────────▶ API
    └── dosage_amount (剂量)

API ──[sourced_from]─────────────▶ Supplier
    ├── supplier_location (供应商位置)
    └── supply_percentage (供应比例)

Supplier ──[provides_to]───────────▶ Manufacturer
    ├── contract_type (合同类型)
    └── delivery_terms (交货条款)

Manufacturer ──[manufactures_at]────▶ Facility
    ├── facility_location (设施位置)
    └── capacity_rating (产能评级)
```

## 六、典型查询场景

### 6.1 供应链追溯查询

```cypher
// 查询: 给定药品，追溯其所有原料供应商
MATCH (d:Drug {name: "Drug X"})-[:contains]->(api:API)
MATCH (api)-[:sourced_from]->(s:Supplier)
MATCH (s)-[:located_in]->(c:Country)
RETURN d.name, api.name, s.name, c.name,
       s.supply_percentage, s.contract_type
```

### 6.2 短缺预警查询

```cypher
// 查询: 监控潜在短缺风险并评估影响
MATCH (se:ShortageEvent {status: "Active"})-[:affects]->(d:Drug)
MATCH (d)-[:contains]->(api:API)
MATCH (api)-[:sourced_from]->(s:Supplier)
WHERE s.supply_percentage > 50
RETURN se, d, api, s, se.severity, se.expected_duration
ORDER BY se.severity DESC
```

### 6.3 供应商评估查询

```cypher
// 查询: 供应商风险评估与绩效评分
MATCH (s:Supplier)-[r:supplies]->(m:Material)
WHERE s.quality_rating < 3.0 OR s.delivery_performance < 80
RETURN s.name, m.name,
       s.quality_rating,
       s.delivery_performance,
       s.certification_status,
       s.geographic_risk
ORDER BY s.quality_rating ASC
```

### 6.4 传播链路查询

```cypher
// 查询: 短缺事件在供应链中的传播链路
MATCH path = (se1:ShortageEvent)-[:propagates_to*]->(se2:ShortageEvent)
WHERE se1.cause = "raw_material_shortage"
RETURN path, se1, se2,
       [r in relationships(path) | r.propagation_delay] as delays
```

## 七、数据源与优先级

| 数据源类型 | 优先级 | 主要用途 | 示例 |
|-----------|-------|---------|------|
| **内部业务系统** | 1 | 采购、生产、库存数据 | ERP, SCM, MES |
| **公共数据库/监管** | 2 | 短缺信息、监管状态 | FDA Drug Shortages, EUDRAF |
| **行业数据源** | 3 | 市场信息、行业趋势 | Pharma publications, Market reports |
| **实时信息** | 4 | 预警、突发状况 | News feeds, Social media, Alerts |

### 标识符映射策略

```
Manufacturer标识符:
├── 主标识符: FDA Establishment Identifier (FEI)
├── 次要标识符: DUNS Number, VAT Number
└── 内部标识符: 供应商主数据编号

Drug/Product标识符:
├── 主标识符: NDC Code (美国), SN (欧洲)
├── 次要标识符: UNII, EAN
└── 内部标识符: SKU,物料编码

Facility标识符:
├── 主标识符: GLN (Global Location Number)
├── 次要标识符: FEI
└── 内部标识符: 工厂编号
```

## 八、与R&D数据关联

**关联策略**：**部分关联**，在API层面关联

| 关联场景 | R&D实体 | 供应链实体 | 关联方式 |
|---------|--------|-----------|---------|
| **API供应商** | Compound (API) | Supplier | Compound.supplied_by |
| **工艺开发** | Manufacturing Process | Facility/Capability | Process.requires_facility |
| **商业化准备** | PCC | Manufacturing Sourcing | PCC.requires_sourcing |

**设计决策**：
- 供应链KG保持相对独立
- 通过**API实体**作为桥接点
- 关联主要用于：商业化决策、工艺转移、外包策略

SupplierPerformance

│   ├── batch_acceptance_rate (批次合格率)
