## 六、典型查询场景

### 6.1 受试者筛选查询

```cypher
// 查询: 查找符合入排标准的受试者
MATCH (s:Subject)
WHERE s.age >= 18
  AND s.ecog_status <= 2
  AND NOT EXISTS (s)-[:has_exclusion_criteria]->(:ExclusionCriteria)
  AND s.screening_status = "Passed"
  AND s.informed_consent_obtained = true
RETURN s.subject_id, s.age, s.gender, s.ecog_status,
       s.eligibility_status
ORDER BY s.screening_date ASC
```

### 6.2 试验进度监控

```cypher
// 查询: 分析试验进度、入组情况、完成率
MATCH (ct:ClinicalTrial)-[:conducted_at]->(s:InvestigationalSite)
WHERE ct.protocol_id = "PROTOCOL-123"
RETURN ct.protocol_id,
       s.site_name,
       count {(s)<-[:enrolled_in]-(ct)} as enrolled_count,
       count {(s)<-[:completed]-(ct)} as completed_count,
       (enrolled_count * 100.0 / ct.target_enrollment) as enrollment_percentage,
       s.site_performance_score
ORDER BY enrollment_percentage DESC
```

### 6.3 多中心数据整合

```cypher
// 查询: 整合多中心数据、进行统计分析
MATCH (ct:ClinicalTrial)-[:conducted_at]->(s:InvestigationalSite)
MATCH (ct)-[:has_arm]->(arm:TreatmentArm)
MATCH (arm)<-[:includes_subject]-(subject:Subject)
MATCH (subject)-[:has_visit]->(v:Visit)
MATCH (v)-[:collected_data]->(data:ClinicalData)
WHERE ct.protocol_id = "PROTOCOL-123"
  AND data.data_quality_score >= 0.8
RETURN arm.arm_name,
       s.site_name,
       count(DISTINCT subject.subject_id) as subject_count,
       collect(data.data_type) as data_types,
       avg(data.completeness) as avg_completeness
ORDER BY subject_count DESC
```

### 6.4 安全性数据分析

```cypher
// 查询: 分析安全数据、生成安全性报告
MATCH (subject:Subject)-[:experienced]->(ae:AdverseEvent)
MATCH (ae)-[:assessed_by]->(sa:SafetyAssessment)
MATCH (ae)-[:associated_with]->(drug:InvestigationalProduct)
WHERE drug.protocol_id = "PROTOCOL-123"
  AND ae.severity IN ["Grade 3", "Grade 4", "Serious"]
  AND sa.causality_category IN ["Related", "Possibly Related]
WITH subject, ae, drug, sa
RETURN subject.subject_id,
       ae.event_type,
       ae.severity,
       sa.causality_category,
       count(ae) as event_count
ORDER BY sa.causality_category DESC, ae.severity DESC
```

### 标识符映射策略

```
ClinicalTrial标识符:
├── 主标识符: NCT Number (美国)
├── 次要标识符: ChiCTR (中国), EudraCT (欧盟), JCTN (日本)
└── 内部标识符: Study Protocol Number

Subject标识符:
├── 主标识符: Subject Unique ID (试验内唯一)
```

## 八、与R&D数据关联

**关联策略**: **部分关联**

临床KG与R&D数据应在关键节点轻量关联，保持领域边界清晰。

| 关联场景 | R&D实体 | 临床实体 | 关联方式 |
|---------|--------|---------|---------|
| **靶点验证** | Target | ClinicalTrial | Target.validation_trial |
| **生物标志物** | Biomarker | Endpoint | Biomarker.used_as_endpoint |
| **安全性关联** | Compound | AdverseEvent | Compound.safety_profile |
| **受试者筛选** | Disease | InclusionCriteria | Disease.inclusion_criteria |

**设计决策**:
```turtle
# 轻量关联：通过属性引用，而非实体合并

# 临床试验引用R&D靶点
:ClinicalTrial
    :investigates_target :Target ;
    :target_id "TARGET-123"^^xsd:string ;
    :target_source "R&D_Database"^^xsd:string .

# 终点引用R&D生物标志物
:Endpoint
    :biomarker_type "Predictive"^^xsd:string ;
    :biomarker_id "BIOM-456"^^xsd:string ;
    :biomarker_source "R&D_Assay"^^xsd:string .

# 不良事件关联R&D化合物
:AdverseEvent
    :suspected_product :Compound ;
    :compound_id "COMPOUND-789"^^xsd:string ;
    :compound_source "R&D_Formulation"^^xsd:string .

# 查询：通过临床数据回溯R&D信息
MATCH (ct:ClinicalTrial)-[:investigates_target]->(target:Target)
WHERE ct.protocol_id = "PROTOCOL-123"
RETURN target.name, target.target_id

# 查询：R&D靶点的临床验证状态
MATCH (t:Target)-[:validated_in]->(ct:ClinicalTrial)
MATCH (ct)-[:has_outcome]->(outcome:TrialOutcome)
WHERE t.target_id = "TARGET-123"
  AND outcome.result = "Positive"
RETURN ct.protocol_id, outcome.result, outcome.evidence_level
```

## 十一、核心发现总结

### 临床领域的独特性

与R&D、供应链、监管领域相比，临床领域具有以下独特性：

1. **受试者中心性**: 受试者既是数据贡献者，也是伦理保护对象
2. **多中心复杂性**: 全球多中心试验涉及标准化、时区、文化差异
3. **时间维度关键**: 试验生命周期长，状态变化频繁，需要时态建模
4. **监管要求严格**: GCP、伦理审查、安全性报告有严格时限要求
5. **数据质量至关重要**: 数据质量直接影响试验结果和监管审批

### 跨领域关联需求

临床领域是R&D和监管合规之间的**关键桥梁**：

```
R&D → Clinical → Regulatory
  ↓        ↓         ↓
Target  Trial  Submission
Safety  Data    Approval
Biomarker Endpoint  Labeling
```

这种定位使得临床KG必须能够：
- 向上对接R&D的靶点、化合物、生物标志物信息
- 向下支撑监管的申报、安全性、有效性证据
